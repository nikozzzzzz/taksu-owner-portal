# Тестирование в Taksu Owner Portal

Этот документ описывает инфраструктуру автоматизированного тестирования для проекта.

## Типы тестов

1. **End-to-End (E2E) тесты**
   - Фреймворк: [Playwright](https://playwright.dev/)
   - Директория: `tests/e2e/`
   - Цель: Полная симуляция действий пользователя в браузере. Проверяет авторизацию, переходы по страницам, отображение графиков и взаимодействие с формами (например, создание заявок).

2. **Unit / Integration тесты**
   - Фреймворк: [Jest](https://jestjs.io/)
   - Директория: `tests/unit/` (в разработке)
   - Цель: Тестирование отдельных утилит, бизнес-логики (например, модуль вычислений `Calculations module` для PPh 26) без запуска реального браузера.

## Запуск тестов

### Локально

```bash
# Запуск Unit-тестов
pnpm test:unit

# Запуск E2E тестов в консольном режиме
pnpm test:e2e

# Запуск E2E тестов с UI Playwright (для отладки)
pnpm exec playwright test --ui
```

> **Примечание:** Для работы E2E тестов локально, необходимо убедиться, что переменная `NEXT_PUBLIC_SUPABASE_URL` в `.env.local` указывает на ваш локальный dev-сервер (`http://localhost:3000`) или на тестовый стенд. Playwright использует эту переменную как `baseURL`.

### В CI/CD (Во время деплоя)

Начиная с версии 0.1.0, тестирование интегрировано в скрипт `deploy.sh`.

**Как это работает:**
1. Выполняется `pnpm build`.
2. Устанавливаются браузеры Playwright (`npx playwright install --with-deps chromium`).
3. Запускается `pnpm test:e2e`. Тесты прогоняются "headless" (в фоне) на сервере.
4. Только если все тесты прошли успешно, процесс PM2 перезапускается (`pm2 restart`) и новая версия становится доступной.
5. Если тесты падают, деплой прерывается, и старая рабочая версия в PM2 продолжает работу без простоев.

## Написание E2E тестов

При добавлении нового функционала, обязательно создайте файл `<feature>.spec.ts` в `tests/e2e/`.

Пример структуры теста:
```typescript
import { test, expect } from '@playwright/test';

test.describe('Название Фичи', () => {
  test.beforeEach(async ({ page }) => {
    // Вход перед каждым тестом
    await page.goto('/login');
    // ... заполнение формы ...
  });

  test('Пользователь может сделать X', async ({ page }) => {
    // Действие
    await page.click('button#do-x');
    // Проверка
    await expect(page.locator('.success-message')).toBeVisible();
  });
});
```

Для E2E тестов мы используем эмуляцию браузера Chromium, чтобы сэкономить ресурсы сервера на CI шаге.
